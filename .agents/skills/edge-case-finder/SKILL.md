# Edge Case Finder スキル

## 目的

日米リードラグ戦略コードの境界値・異常系・例外パターンを体系的に列挙し、ルックアヘッドリーク・計算崩壊・フォールバック予期せぬ発動を未然に防ぐ。

## 検査観点

### 1. ローリング窓の境界

- **窓サイズ不足**: 行数 < 窓サイズ時の挙動。`min_periods` の設定是否、NaN伝播の有無
- **当日行の混入**: 相関窓 `all_returns[window_start:current_index]` が当日行を含んでいないか（`src/leadlag/core/signal.py`）
- **窓の境界オフバイワン**: `window_start` の計算が `current_index - window_size` か `current_index - window_size + 1` か
- **先頭データのフォールバック**: `_prepare_residual_prior` の先頭1260行フォールバックが発動する構成になっていないか

### 2. PIT（Point-in-Time）ビニング

- **未来情報の混入**: PIT三分位の境界計算に当日行が含まれていないか（`check_pit_binning_lookahead`）
- **ビニング境界値**: 三分位の閾値が `<=` か `<` か。境界値の銘柄が複数ビンにまたがらないか
- **PITデータの欠損**: 過去データが一部欠損している場合のビニング挙動

### 3. NaN / Inf 伝播

- **yfinanceの欠損データ**: 米国ETFの一部銘柄が取得失敗した場合の伝播
- **ゼロ除算**: ベータ計算・正規化で分母がゼロになるケース
- **Infのキャッシュ混入**: `_PRODUCTION_SIGNAL_CACHE` にInfが含まれた状態でキャッシュされるリスク
- **空DataFrame**: 銘柄数が0または1の場合の相関行列計算

### 4. フォールバック（現行: フラットポジションのみ。V1フォールバックは2026-07廃止）

- **発動条件**: gapデータ欠損（`mu_gap`/`Omega_gap` なし）または numerical audit 失敗（`fallback_on_audit_failure=True` 時）で w_final=0。予期せぬ発動がないか（`production_v2.py`）
- **PIT履歴不足**: `history_count < pit_rolling_window` 時の `fallback_multiplier` 適用が意図通りか
- **フォールバック発動率**: バックテスト全体で発動率が異常に高くないか（5%超は要調査）。一度フラット化した後に正常復帰するか

### 5. ポートフォリオ制約

- **net exposure ±0.05 違反**: 極端なシグナルで制約を逸脱しないか
- **gross ≤ 2.0 違反**: RuleD 適用後に制約を満たすか
- **全銘柄ゼロシグナル**: シグナルが全てゼロの場合のポートフォリオ構築挙動
- **単一銘柄への集中**: シグナルが1銘柄に極端に集中する場合のリスク制御

### 6. 日付・時刻の境界

- **年末年始**: 日本の年末年始（12/31〜1/3）の営業日処理
- **米国祝日**: 米国休場でUS列がNaNの場合の挙動
- **日付アラインメント**: `df_exec`の行tとD_t/D_{t+1}の対応が崩れていないか
- **うるう年**: 2/29の処理

### 7. データ取得の異常

- **yfinanceハング**: ダウンロードがタイムアウトしない（`docs/スタック再発防止策.md` P2参照）
- **ファイルロック競合**: `cache.py` の fcntl.flock がブロッキングしないか
- **重複データ**: 同一日付のデータが重複して格納された場合の挙動

### 8. Copula相関ブレンドの境界

- **t-copula nu収束**: 自由度νの最適化が収束しない場合の挙動。`_t_copula_neg_loglik` の正定値性
- **特異行列**: copula log-likelihood計算で相関行列が特異になるケース。jitter・pseudo-inverseフォールバックの発動条件
- **ブレンド後NaN**: Pearson相関とcopula相関のブレンドでNaNが伝播しないか
- **ストレス判定境界**: 動的ブレンドのストレス閾値（vol_ratio >= 1.5）の境界値でブレンド重みが急激に変化しないか
- **copula窓の当日行混入**: copula推定に使用するローリング窓が当日行を含んでいないか

### 9. MinVar weight最適化の境界

- **Sigma_YY特異**: 予測共分散`Omega_gap`が特異（正定値でない）場合の`build_weights_minvar`の挙動
- **α=1.0の退化**: 純最小分散（α=1.0）でシグナル情報が無視され、Sharpeが大幅劣化する現象（実測: -0.40）
- **α=0.0の等価性**: α=0.0でsignal比例weightと完全一致するか
- **バスケット内1銘柄**: ロングまたはショートバスケットが1銘柄のみの場合のminvar解
- **Omega_gapゼロ行列**: 全要素がゼロに近い場合のminvar解の安定性

### 10. Macro Confidenceの境界

- **マクロデータ欠損**: yfinanceでUSDJPY・CLF・TNXのいずれかが取得失敗した場合の挙動
- **サプライズゼロ**: マクロサプライズが全てゼロの時、scale=1.0になりシグナル無変化となるか
- **kappa過大**: kappaが大きすぎてscaleが発散しシグナルが過小縮小されないか

## 実行手順

1. 対象コードを読み込み、上記10観点を順にチェック
2. 検出したエッジケースをリスト化（観点番号・ファイル・行番号・リスクレベル付き）
3. リスクレベル: **Critical**（リーク・計算崩壊） / **Warning**（異常時の挙動不安） / **Info**（理論上可能だが発生確率低）
4. Critical には最小限の修正案を添える
5. `ComplianceAuditor`（`src/leadlag/compliance/auditor.py`, `v2_auditor.py`）の監査項目との対応を確認

## 出力形式

```
## Edge Case Report

| # | 観点 | ファイル:行 | リスク | 内容 | 修正案 |
|---|------|------------|--------|------|--------|
| 1 | 1-窓境界 | signal.py:45 | Critical | ... | ... |
```

## 注意事項

- このスキルは検出・列挙が主目的。修正実装は別途行う
- `ComplianceAuditor` の監査項目を無効化する修正は禁止
- 検出結果は必ずレポート化し、`reports/` 配下に保存すること
