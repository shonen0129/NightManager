# 日米業種リードラグ市場中立戦略：モデル要点・計算仕様まとめ（LLM改善検討用）

本ドキュメントは、日米の業種（セクター）ETF間のリードラグ関係を利用した「日米業種リードラグ市場中立戦略」について、他のAI（LLM）に改善案を検討させるためのインプットとして、モデルの要点、数理的計算方法、パラメータ仕様を体系的にまとめたものである。

---

## 1. 戦略概要と基本哲学

### 1.1 リードラグ仮説（取引時間帯の非同期性）
米国市場（東部時間 9:30～16:00）と日本市場（日本時間 9:00～15:00）の間には取引時間のズレ（非同期性）が存在する。米国市場の当日終値までに織り込まれたグローバルな業種別トレンド（情報）の一部は、翌営業日の日本市場における同業種のセクターETFの日中リターン（寄付き～大引け）に遅れて波及する。このクロスマーケットの情報伝播の遅延を統計的に検出し、絶対収益を獲得する。

### 1.2 投資ユニバース
*   **米国側（予測特徴量 / 情報源）**: $N_{US} = 15$ 銘柄
    *   主要セクターETF（Select Sector SPDR） 11本 (XLB, XLC, XLE, XLF, XLI, XLK, XLP, XLRE, XLU, XLV, XLY)
    *   スタイル・ファクター型ETF 4本 (MTUM, VLUE, IUSG, USMV)
*   **日本側（取引対象）**: $N_J = 17$ 銘柄
    *   NEXT FUNDS TOPIX-17業種別ETFシリーズ（1617.T～1633.T）
*   **結合ユニバース**: 米国15銘柄＋日本17銘柄＝全32銘柄の次元で相関構造を扱う。

### 1.3 執行のタイムラインと保有期間
*   **シグナル算出**: 米国市場終値確定後（日本時間早朝5:00～6:00）に基本予測シグナルを生成。
*   **ポジション構築（エントリー）**: 翌営業日の寄付き直後の価格ノイズを回避しつつリードラグ効果を最大化するため、**「9:10」**時点の価格でポジションを構築。
*   **ポジション決済（イグジット）**: 同日の大引け（15:00～15:30）にすべて成行または引けVWAP等で決済。原則としてオーバーナイトポジションは保有しない（日中のみの取引）。

## 2. 現行本番モデル：Production Residual-BLPX-RA v2

現行の正式本番モデルは、残差化された被予測ターゲットに対し、Ridge正則化と構造的縮小（Structured Shrinkage）を施した線形予測器（BLPX）に、予測分布から得られるリスク調整スコア（$\mu/\sigma$）と、実績インフォメーション・レシオ（IR）による動的グロス調整（RuleD）を組み合わせたものである。

### 2.1 被予測変数（日本側リターン）のTOPIX残差化
日本セクターETFの変動に含まれる日本市場全体の共通要因（TOPIX）の影響を除去し、固有のアルファを予測するため、日本側リターンをあらかじめ残差化する。
1.  日本業種ETF $j$ の Close-to-Close リターン $r_{j,\tau}^{\text{cc}}$ と TOPIX リターン $r_{\text{TOPIX},\tau}^{\text{cc}}$ について、直近60日のローリング OLS でベータ $\beta_{j,t}$ を推定する。
2.  ルックアヘッド（将来情報のリーク）を防ぐため、1営業日ラグをとったベータ $\beta_{j,\tau-1}$ を用いて残差リターン $y^{\text{res}}_{j,\tau}$ を計算する：
    $$y^{\text{res}}_{j,\tau} = r_{j,\tau}^{\text{cc}} - \beta_{j,\tau-1} \times r_{\text{TOPIX},\tau}^{\text{cc}}$$
3.  この $y^{\text{res}}$ を日本側のターゲット系列としてモデル推定に使用する。

### 2.2 共分散行列の推定と正則化
1.  **クリッピング（Winsorization）**: 過去 $L_{\text{blp}} = 252$ 営業日の結合リターン行列 $R_{\tau} = [X_{\tau}, Y^{\text{res}}_{\tau}]$（32次元）に対し、極端な異常値の影響を抑えるため、平均 $\pm 3.0\sigma$ でWinsorize処理を行う。
2.  **指数加重共分散（EWMA）**: 半減期 $h = 45$ 営業日の指数加重を用いて、32×32の結合相関行列 $C$ を算出する。
    $$C = \begin{pmatrix} C_{XX} & C_{XY} \\ C_{YX} & C_{YY} \end{pmatrix}$$
    ここで $X$ は米国15銘柄、$Y$ は日本残差17銘柄。
3.  **ブロック別収縮（Shrinkage）**: 各ブロックに以下のパラメータ（本番設定値: $\alpha_{xx}=0.50, \alpha_{yx}=0.05, \alpha_{yy}=0.50$）を用いて正則化を適用する。
    $$\Sigma_{XX} = (1 - \alpha_{xx}) C_{XX} + \alpha_{xx} I_{15}$$
    $$\Sigma_{YX} = (1 - \alpha_{yx}) C_{YX}$$
    $$\Sigma_{YY} = (1 - \alpha_{yy}) C_{YY} + \alpha_{yy} I_{17}$$
4.  **Ridge正則化**: 米国側共分散行列 $\Sigma_{XX}$ に対角平均値基準の Ridge 正則化（パラメータ: $\rho=0.01$）を適用して、逆行列計算を安定化させる。
    $$A = \Sigma_{XX} + \rho \cdot \text{mean}(\text{diag}(\Sigma_{XX})) \cdot I_{15}$$

### 2.3 構造的縮小（Structured Shrinkage）
基本BLPX予測行列 $B_{\text{blp}} = \Sigma_{YX} A^{-1} \in \mathbb{R}^{17 \times 15}$ に対し、ノイズの低減と事前知識の注入のために多ターゲットTikhonov正則化を行う。
$$A_{\text{tikh}} = \Sigma_{XX}^{\text{reg}} + (\rho \cdot \bar{d} + \lambda_{\text{pca}} + \lambda_{\text{sector}}) I$$
$$B_{\text{struct}} = (\Sigma_{YX}^{\text{reg}} + \lambda_{\text{pca}} B_{\text{pca}} + \lambda_{\text{sector}} M_{\text{sector}}) \, A_{\text{tikh}}^{-1}$$
*   **本番設定パラメータ**: $\lambda_{\text{pca}} = 0.10, \lambda_{\text{sector}} = 0.60$
*   **PCA事前分布 $B_{\text{pca}}$**: 後述する部分空間正則化PCA（6成分）から算出される予測伝播行列。
*   **セクター事前分布 $M_{\text{sector}}$**: 日米の類似業種を1対1で対応させた固定マッピング行列。
*   ※ Frobeniusノルムスケーリングは2026-07のバックテスト比較で非採用（raw priorsが優位: net Sharpe 8.35 vs 8.03）。Tikhonov定式化では共分散構造を通じた正則化が行われるため、凸結合で必要なノルム正規化は不要。

この構造化予測行列を用いて、日本側の予測Zスコア期待値を算出する：
$$\hat{z}_{J,t+1} = B_{\text{struct}} z_{U,t}$$

### 2.4 条件付き予測分散による確信度調整（Confidence Weighting）
予測値のばらつき（確信度）を考慮するため、条件付き共分散行列 $\Sigma_{Y|X}$ から予測分散を求め、シグナルを調整する。
1.  条件付き共分散行列の対角成分（予測分散）を抽出する：
    $$\Sigma_{Y|X} = \Sigma_{YY} - \Sigma_{YX} A^{-1} \Sigma_{XY}$$
    $$\text{pred\_var}_j = \text{diag}(\Sigma_{Y|X})_j$$
2.  確信度感応度 $\beta_{\text{conf}} = 0.25$ を用いて、予測値 $\hat{z}_{J,t+1,j}$ をリスク調整スケーリングし、最大値 $\pm 5.0$ にクリップする：
    $$z_{\text{conf}, j} = \frac{\hat{z}_{J, t+1, j}}{(\text{pred\_var}_j)^{\beta_{\text{conf}}}}$$

### 2.5 翌朝のリアルタイム・ギャップオープン補正（翌営業日 9:10 時点）
日本市場の寄付き時の夜間ギャップリターン $GapOpen_{j,t+1}$ には、米国情報の織り込みや日本全体のシステミックショックが含まれるため、寄付き後にシグナルを再補正する。
1.  日本側の寄付きギャップ $GapOpen_{j,t+1}$ に対し、TOPIX夜間リターンに対するローリングベータ $\beta_{j,t}$ （窓幅 60日）を用いて残差化する：
    $$GapOpen\_syst_{j,t+1} = \beta_{j,t} \times TOPIXNight_{t+1}$$
    $$GapOpen\_idio_{j,t+1} = GapOpen_{j,t+1} - GapOpen\_syst_{j,t+1}$$
2.  補正後のギャップリターン $GapOpen\_filt_{j,t+1}$ を計算する：
    $$GapOpen\_filt_{j,t+1} = c \times GapOpen\_idio_{j,t+1} + (c - b) \times GapOpen\_syst_{j,t+1}$$
    *   本番設定値: ギャップ係数 $c = 0.70$, TOPIXベータ係数 $b = 0.60$
3.  確信度調整後シグナル $z_{\text{conf},j}$ に、直近20日のローリング標準偏差 $\sigma^{20}_{j,t}$ を乗じて期待リターンに換算し、ギャップリターンで除算（引算）して最終取引シグナル $s_{j,t}$ を決定する：
    $$s^{\text{vol}}_{j,t+1} = z_{\text{conf},j} \times \sigma^{20}_{j,t}$$
    $$s_{j,t} = \frac{1 + s^{\text{vol}}_{j,t+1}}{\max(1 + GapOpen\_filt_{j,t+1}, 0.1)} - 1$$

## 3. ポートフォリオ構築と動的リスク管理

### 3.1 銘柄ランキングと選定
各銘柄のギャップ調整済み予測分布から予測期待値 $\mu_{\text{gap},j,t}$ と予測標準偏差 $\sigma_{\text{gap},j,t} = \sqrt{\Omega_{\text{gap},jj,t}}$ を算出し、以下のリスク調整スコア（$\text{score}_{j,t}$）でランキングを行う。
$$\text{score}_{j,t} = \frac{\mu_{\text{gap},j,t}}{\sigma_{\text{gap},j,t}}$$
*   このスコアに基づき、上位 30%（17銘柄中 5銘柄）をロング候補 $L_{t+1}$、下位 30%（5銘柄）をショート候補 $S_{t+1}$ とする。

### 3.2 基本ウェイト配分（baseline_style）
スコアの中央値 $\text{median}(\text{score}_{\cdot,t})$ からの乖離幅に比例したシグナル加重ウェイトを構成する（$\tilde{s}_{j,t} = \text{score}_{j,t} - \text{median}(\text{score}_{\cdot,t})$）。
$$w'_{j,t+1} = \frac{\tilde{s}_{j,t}}{\sum_{k \in L_{t+1}} \tilde{s}_{k,t}} \times \frac{G_t}{2}, \quad j \in L_{t+1}$$
$$w'_{j,t+1} = \frac{-\tilde{s}_{j,t}}{\sum_{k \in S_{t+1}} (-\tilde{s}_{k,t})} \times \left(-\frac{G_t}{2}\right), \quad j \in S_{t+1}$$
$$w'_{j,t+1} = 0, \quad \text{それ以外}$$
これにより、ドルニュートラル（$\sum w_j = 0$）かつグロス $G_t$ のロング・ショートポートフォリオが構築される。

### 3.3 動的グロス調整（RuleD）
予測される ex-ante インフォメーション・レシオ（予測IR）の実績分位に基づき、マーケット全体のシグナル信頼度が低いときにリスクを自動削減する。
*   直近252営業日の ex-ante ポートフォリオ IR の実績履歴の33.3%点（$P_{33}$）および66.7%点（$P_{67}$）を閾値とする。
*   当日の予測 IR $\hat{IR}_t$ が属する位置に応じて、グロス乗数 $m_t$ を乗じる：
    *   **Low** ($\hat{IR}_t \leq P_{33}$): グロス乗数 **0.75** (適用グロス: **150%**)
    *   **Medium / High** ($\hat{IR}_t > P_{33}$): グロス乗数 **1.00** (適用グロス: **200%**)
*   最終ウェイト: $w_{j,t+1} = w'_{j,t+1} \times m_t$

## 4. フォールバックおよびベンチマークモデルの仕様

システム上の安全監査（Safety Audit）に失敗した場合やデータ欠損時には、自動的に以下のフォールバック階層に従って代替モデルが作動する。

### 4.1 第1フォールバック: Residual-BLPX v1
現行 v2 の確信度調整前・動的グロス調整（RuleD）前の Residual-BLPX (v1) シグナルによるランキングおよび固定グロス（200%）ウェイト。ギャップ分布データ欠損時の第1防衛線となる。

### 4.2 最終フォールバック / ベンチマーク: PCA-Ensemble (Sector Relative Ensemble)
より頑健な「部分空間正則化PCA」に基づくアンサンブルモデル。
*   **Raw-PCA** (TOPIX残差化なしの生リターンに基づくPCA予測) と **Residual-PCA** (TOPIX残差化されたターゲットに対するPCA予測) の予測シグナルを各 50% ずつアンサンブル。
*   **部分空間正則化**: ローリング相関行列に対し、長期（2010～2014年）データから構築した「経済的ファクター（グローバル、国スプレッド、シクリカル、為替、エネルギー、インフレ）からなる 6次元の事前部分空間 $C_0$」へ一定比率（$\lambda_{\text{reg}}=0.75$）で正則化縮小を行い、推定誤差を抑制する。

## 6. 改善アプローチ・論点

この戦略をさらに改善するために、以下のような切り口からブレインストーミングや数理モデルの再考、アルゴリズムの提案を促すことができる。

### 【論点 1】シグナル予測モデルの高度化・非線形化
*   **現状**: 現在はRidge回帰と事前分布との凸結合（線形モデル）を採用。
*   **問いかけ**: 日米間の情報伝播に非線形な関係や業種特有の閾値効果（例えば、米国半導体株が急激に動いた時だけ日本の電機精密へ伝播するなど）が存在する場合、どのような手法（Lasso、ElasticNet、Kernel Ridge、もしくはLightGBM/XGBoostなどの決定木アンサンブル、あるいは浅いニューラルネットワークなど）を用いて過学習を抑制しつつ非線形性を捉えるべきか？
*   **制約**: $L=252$ 営業日のようにサンプル数が比較的小さく、銘柄数が32次元と多重共線性が高いため、過学習（オーバーフィッティング）対策が必須。

### 【論点 2】事前部分空間およびファクター定義の改良
*   **現状**: 2010～2014年のデータから定義した 6つのマクロ/スタイルファクター（グローバル、国スプレッド、シクリカル、為替、エネルギー、インフレ）を事前部分空間として正則化に使用している。
*   **問いかけ**:
    1.  近年（2020年以降）の金利上昇やAI半導体ブーム、地政学リスクの高まりなどを考慮し、新たな事前ファクター（例：半導体/テックファクター、サプライチェーンファクター、地政学リスク等）を追加・修正すべきか？
    2.  あるいは、静的なラベル定義ではなく、マクロ指標（USD/JPY、WTI原油、米国10年金利など）に対する時系列感応度（ローリングベータ）を用いて、事前部分空間を動的にアップデートする手法は有効か？

### 【論点 3】リアルタイム・ギャップオープン補正の高度化
*   **現状**: 日本の朝9:00～9:10までに生じる夜間ギャップリターンを、TOPIX夜間リターンに対するローリングベータでシステミックと固有に分け、固有部分を 0.70倍、システミック部分を 0.10倍する（$c=0.70, b=0.60$）という経験的な静的ルールで補正している。
*   **問いかけ**:
    1.  朝のギャップオープンにおける米国市場の終値での織り込みの「達成度」や「オーバーシュート（戻り）」を、朝時点のドル円の為替動向やシカゴ日経先物の乖離、米国VIX等を用いて動的（時変パラメータ）に補正するアプローチはないか？
    2.  カルマンフィルター等を用いて、夜間のグローバルショックの強さに応じて補正係数 $c$ や $b$ をリアルタイムに最適化することは可能か？

### 【論点 4】動的サイズ管理（RuleD）の最適化
*   **現状**: 過去252営業日の予測IRの実績値の33.3%点を下回る局面で、グロスエクスポージャーを 150% に削減（通常は 200%）。
*   **問いかけ**:
    *   3分位点による離散的なステップ関数ではなく、予測IRや予測ポートフォリオボラティリティ、あるいは市場全体のストレス度合い（VaR/ES等）に基づいて、グロスエクスポージャーを連続的かつ動的にスケーリングする数理モデル（例えばKelly基準の変形や、予測シャープレシオに比例したポジション制限）はどのように設計できるか？

### 【論点 5】流動性制約・取引コスト（スリッページ）モデルの導入
*   **現状**: TOPIX-17 ETFは東証のマーケットメイク制度（提示金額3,000万円、スプレッド0.25%以内等）があるが、主要指数ETFに比べると流動性が低い。
*   **問いかけ**:
    *   ブローカーの約定不利（スリッページ）や板の薄さを考慮し、ファンドの運用総額（AUM）が拡大した際にもスプレッド負けしないよう、取引コストを明示的に定式化した平均分散最適化（Mean-Variance Optimization with Transaction Costs）をウェイト配分に組み込むべきか？

### 【論点 6】代替データ・ネットワークファクターの導入
*   **現状**: ETFの時系列価格データのみからシグナルを生成している。
*   **問いかけ**:
    *   サプライチェーンデータ、特許データの共起関係、あるいはEDINETなどの開示書類から構築した「企業間ネットワーク・業種間トポロジー」の情報を、結合共分散行列の事前分布（セクター事前分布 $B_{\text{sector}}$）や事前部分空間ベクトルに定量的に反映させるアプローチはないか？
