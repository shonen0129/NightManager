## C1. 多重試行補正フレームワーク（Deflated Sharpe + Trials Registry）

### 理論

Deflated Sharpe Ratio（Bailey & López de Prado 2014）:

```
DSR = Φ( (SR_hat − SR₀) · √(T−1) / √(1 − γ₃·SR_hat + (γ₄−1)/4·SR_hat²) )

SR₀ = √(V[SR_n]) · ( (1−γ)·Φ⁻¹(1−1/N) + γ·Φ⁻¹(1−1/(N·e)) )
```

- SR_hat: 候補構成の日次 Sharpe（非年率）、T: 標本日数、γ₃/γ₄: リターンの歪度・尖度
- N: **実効試行回数**、V[SR_n]: 試行間 Sharpe の分散（registry から推定）
- γ: Euler-Mascheroni 定数（0.5772）
- 判定基準: **DSR ≥ 0.95 を本番昇格の必要条件**とする

### 実効試行回数 N の定義（本リポジトリ固有の問題への回答）

archive の約30実験をどうカウントするか:

1. **原則**: 「同一ヒストリー上で Sharpe を確認した構成」1つ = 1試行。
   パラメータグリッドは格子点数をそのまま数える（グリッド9点 = 9試行）
2. **相関補正**: 試行同士のリターン系列相関が高い場合、López de Prado の
   クラスタリング（試行リターン相関行列の固有値分析）で実効数 N_eff に縮約。
   `reports/pbo/returns_matrix.npy` が既にあるため同じ枠組みで実装可能
3. **過去分の初期値**: archive/experiments 約30本 + configs/archive 15本 + research 10本
   ≈ 55試行を初期登録とする。正確な復元は不可能なので**保守側（多め）に見積もる**

### Trials Registry 仕様

`reports/trials_registry.csv`（追記専用、実験実行時に必ず1行追加）:

| 列 | 内容 |
|----|------|
| date | 実験日 |
| experiment_id | スクリプト名 or レポート名 |
| n_configs_evaluated | その実験で Sharpe を確認した構成数（グリッド点数含む） |
| best_net_sharpe | 最良構成の net Sharpe |
| adopted | 採用/不採用 |
| returns_file | 日次リターン系列の保存パス（相関補正用、任意） |

運用ルール:
- backtest-report skill の必須項目に「registry 追記済み」チェックを追加
- DSR 計算スクリプト `tools/validation/compute_deflated_sharpe.py` を新設し、
  registry から N・V[SR_n] を自動集計

### PBO との関係

既存の `reports/pbo/`（CSCV）は「構成選択の頑健性」、DSR は「多重性補正後の有意性」で
相補的。昇格判定は **DSR ≥ 0.95 かつ PBO ≤ 0.2 の AND 条件**を推奨。

## C2. シャドー運用 → 本番昇格の統計的判定基準

### 判定の3要素と検定

**前提**: シャドー期間は最低 60 営業日（下記検出力の根拠を参照）。

#### (1) 実装整合性（シャドー vs バックテスト同日再現）

- 指標: 同一日のシャドー重みとバックテスト再計算重みの相関・最大絶対乖離
- 基準: **日次重み相関 ≥ 0.99 が20営業日連続**（乖離原因が特定・文書化済みの場合を除く）
- これは統計検定ではなく決定論的整合チェック。fail は即調査

#### (2) 性能整合性（シャドー実績 vs バックテスト期待）

- 検定: シャドー日次ネットリターン r_t とバックテスト同日リターン b_t の差
  d_t = r_t − b_t に対する 片側 t 検定（H₀: E[d] ≥ −c、c = 想定執行コスト差 2bps/日）
- tracking error: `TE = std(d_t)·√252` を報告し、**TE ≤ バックテスト年率ボラの15%** を基準
- 60日で検出可能な効果量: d の日次標準偏差を σ_d とすると、
  検出力80%・α=5%で検出可能な平均差 ≈ 0.32·σ_d。σ_d が大きく検出力不足の場合は
  期間延長（90日）を機械的に要求する

#### (3) IC 整合性（シグナル品質の維持）

- シャドー期間の日次 rank IC の平均を、バックテスト同長ローリング IC 分布の
  経験分位と比較。**シャドーICがバックテストIC分布の下位10%タイルを下回らない**こと
- 補助: 符号検定（IC > 0 の日数比率がバックテストと二項検定で有意差なし）

### 自動判定の実装仕様

`tools/validation/monitor_residual_blpx_shadow_performance.py` を拡張:

1. 上記 (1)–(3) を計算し、`shadow_runs/<run>/promotion_report.md` に PASS/FAIL 表を出力
2. 総合判定: (1) AND (2) AND (3) すべて PASS → 「昇格可」、
   1つでも FAIL → 原因分類（データ差 / 執行差 / モデル差）を必須記入の上「保留」
3. 判定ロジックはユニットテスト対象とする（合成データで PASS/FAIL 両方を検証）

### 統計的注意

- 60日の Sharpe 点推定は無意味（SE ≈ 1.6年率）。**シャドーで Sharpe を判定基準にしない**こと。
  判定はあくまで「バックテストとの整合」であり「絶対性能」ではない
- 多重比較: 3検定の AND 条件は保守側（family-wise error を上げない方向）なので補正不要
