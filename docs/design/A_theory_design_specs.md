## A1. ギャップ調整公式の理論的整理

### 現状の形式化

ターゲットは 9:10→大引けリターン r_910。恒等式:

```
(1 + r_cc) = (1 + gap)(1 + r_910)
⇒ E[r_910 | US, gap] = (1 + E[r_cc | US, gap]) / (1 + gap) - 1
```

現行実装（`core/signal.py:140-147`）は `E[r_cc | US]`（gap非条件付き）を分子に使い、分母を
`1 + gap_filt`、`gap_filt = c·gap_idio + (c−b)·gap_syst`（c=0.70, b=0.6）に置換している。

### 理論的解釈（導出）

1次近似 `E[r_910] ≈ E[r_cc|US] − gap_filt` において、真の条件付き期待値は

```
E[r_cc | US, gap] = E[r_cc | US] + β_rev_cc · (gap_idio − E[gap_idio | US])
```

恒等式 r_cc ≈ gap + r_910 より β_rev_cc ≈ 1 + β_rev_oc（実測: β_rev_oc ≈ −0.23、β_rev_cc ≈ +0.73）。
これを代入すると:

```
E[r_910 | US, gap] ≈ E[r_cc | US] + (1 + β_rev_oc)·gap_idio − gap_idio − (係数)·gap_syst
                   = E[r_cc | US] + β_rev_oc · gap_idio − (係数)·gap_syst
```

つまり**理論的に正しい gap_idio 係数は −β_rev_oc ≈ 0.23 ではなく、(1 − (1+β_rev_oc)) = −β_rev_oc … を引く形、
すなわち有効控除係数は 1 + β_rev_oc ≈ 0.77**。現行の c=0.70 はこれに近く、
`sprint_gap_reversal` の「明示補正は二重カウント」という結論は本導出と整合する
（0.70 + 0.23 = 0.93 は 0.77 を大きく超えるため過剰調整 → Sharpe劣化、実験結果と一致）。

### 実装仕様（残作業）

- **固定 c=0.70 を理論値 `c_t = 1 + β_rev_oc(t)` のローリング推定（252日、shift(1)）+ 事前値0.70へのシュリンク**に置換する実験:
  `c_t = (1−λ)·0.70 + λ·(1 + β_rev_oc_hat(t))`、λ ∈ {0.25, 0.5, 0.75} の3点のみ試行
- β_rev_oc は必ず r_910（または r_oc）に対して推定（r_cc は禁止 — 機械的正相関）
- gap_syst 側係数 b も同型で `b_t = c_t − (1 + β_rev_syst(t))` として整合させる

### 過学習ガード / 不採用判定

- λの3点以外はグリッドしない。walkforward OOS net Sharpe が baseline −2% を下回れば不採用
- Deflated Sharpe（試行3回追加として補正）をレポートに含める
- 不採用でも `reports/` に記録（gap_reversal_correction_report.md と相互参照）

## A2. BLP構造化シュリンケージのベイズ的正当化

### 現状の閉形式のベイズ解釈（導出）

`_solve_tikhonov` は実質次を解く:

```
B* = argmin_B ||Σ_YX − B·Σ_XX||² + λ_pca||B − B_pca||² + λ_sector||B − M_sector||² + ρ·d̄||B||²
```

これはガウス尤度 vec(Σ_YX) ~ N((Σ_XX⊗I)vec(B), σ²I) と、独立なガウス事前分布

```
B ~ N(B̄, τ²I),  B̄ = (λ_pca·B_pca + λ_sector·M_sector) / (λ_pca + λ_sector + ρ·d̄)
```

の下での MAP 推定に一致する。**Robust PCA 実験の失敗はこの事前平均 B̄ の構造
（セクター知識 + PCA部分空間）を破壊したため**であり、本解釈から予見可能だった。

### 実装仕様: Empirical Bayes によるλ推定

現在 λ_pca, λ_sector, ρ は固定。周辺尤度最大化で自動決定する:

1. 各時点 t で、直近 blp_window 日の (X, Y) に対し、evidence
   `p(Σ_YX | λ) = ∫ p(Σ_YX | B) p(B | λ) dB`（ガウスなので閉形式）を計算
2. λ = (λ_pca, λ_sector, ρ) を粗いログスケール格子（各3点、計27点）で evidence 最大化
3. 推定λは20日ごとにのみ更新（安定化）し、shift(1) で適用（リーク防止）

### 過学習ガード / 不採用判定

- 格子は27点固定、拡張禁止。λ時系列の分散が大きい（CV > 1）場合は固定λに戻す
- OOS net Sharpe と IC を baseline と比較。改善 < +1% なら「複雑性に見合わない」として不採用
- `ComplianceAuditor` の `check_pit_binning_lookahead` 相当のチェックをλ更新にも追加すること

## A3. PITビニング / RuleD 動的グロスの最適設計

### 理論的整理

RuleD は「ex-ante IR の percentile p → グロス乗数 m」の写像 g(p) の階段関数版
（3分位、0.75/1.0/1.0）。理論的最適形は Kelly 比例: `m*(p) ∝ E[IR|p] / Var[IR|p]`。
IR履歴 ~2500日で分位あたり有効標本 ~800日だが、独立な「レジーム」の数はずっと少ない
（IRの自己相関を考慮した有効標本 ≈ 2500 / (1 + 2Σρ_k) を先に測定せよ）。

### 実装仕様

1. **診断**: PIT percentile p_t と翌日実現ネットリターンの関係を isotonic regression で
   ノンパラ推定し、単調性と形状を可視化（`reports/` に図示）。これだけで
   現行の Low=0.75x が妥当か（下位tertileで本当にE[r]が低いか）判定できる
2. 形状が滑らかな単調なら 2パラメータのロジスティック形
   `g(p) = m_lo + (m_hi − m_lo)·σ((p − p0)/s)` を提案。ただし **p0, s は
   前半期間（2015–2020）で固定推定し、後半（2021–）はOOS検証のみ**
3. 形状が非単調・不安定なら現行3分位を維持（変更しないことが最適解）

### 過学習ガード / 不採用判定

- パラメータは最大2個。乗数上限は 1.0 のまま（レバレッジ増加方向の変更は別途リスク承認が必要）
- OOS期間で MDD が baseline を悪化させたら即不採用
- isotonic 診断で単調性が確認できない場合はステップ2に進まない

## A4. アンサンブル重みの理論的最適化

### 導出

成分シグナル z_i の IC ベクトル μ_IC と IC 誤差共分散 Σ_e に対し、
シャープ最大化の最適重みは `w* ∝ Σ_e^{-1} μ_IC`（Grinold-Kahn / シグナルのMarkowitz）。

現行は (a) 静的重み、(b) meta-learning（5特徴のロジスティック、`sector_relative_ensemble_blp_enhanced.py:869-958`）が混在。
meta-learning は訓練標本 ≥100・特徴5・日次再学習で **分散が大きく理論的に不利**。

### 実装仕様

1. 各成分の日次ICを計算し、ローリング（504日、shift(1)）で μ_IC, Σ_e を推定
2. `w_t = (1−δ)·w_static + δ·normalize(Σ_e^{-1} μ_IC)`、シュリンク δ ∈ {0.25, 0.5} の2点のみ
3. Σ_e は Ledoit-Wolf シュリンク推定（成分数4なので条件は良いが慣例として）
4. **meta-learning は本比較で優位を示せなければ deprecate**（コード削除は別PR）

### 過学習ガード / 不採用判定

- δ 2点のみ。w_t の日次変動がターンオーバーを +5% 以上増やすなら平滑化（EWMA 20日）を強制
- OOS net Sharpe 改善 < +1% なら不採用、静的重み維持

## A5. Bayesian BLPX の Kalman 定式化の診断と修正

### 診断（現行実装 `bayesian_blpx.py:99-129` の問題点）

状態空間モデルとして書くと:

```
状態:   B_t = B_{t-1} + η_t,        η_t ~ N(0, Q)
観測:   B̂_t = B_t + ε_t,           ε_t ~ N(0, R)   （B̂_t = rolling BLP 推定）
```

問題点3つ:

1. **Q の推定に `diff(rolling推定)` を使用** — rolling 窓（blp_window=252日）の
   重複により B̂_t の変化は強く自己相関し、`Var(ΔB̂)` は Q + 2R/window の混合。
   Q を系統的に過大推定 → gain 過大 → 過剰適応
2. **R を per-asset の予測誤差分散で代用** — 予測誤差 y − B̂z は観測ノイズ ε ではなく
   モデル残差（irreducible noise 含む）。R の過大推定
3. per-asset スカラー gain は B の行ごとの単純平均で、要素ごとの情報を捨てている

### 修正仕様

1. Q の推定: 重複のない間隔（window 日おき）の ΔB̂ から
   `Q = max(0, Var(ΔB̂_non-overlap) − 2·R̂)` として bias 補正、または EM で同時推定
2. R の推定: rolling 推定の標本分散理論値 `R̂ ≈ Var(residual)/window · (Σ_XX^{-1}の対角)` を使用
3. まず**スカラーゲイン + 正しいQ/R** で baseline（現行 ic モード）と比較。
   per-element 化はスカラー版が優位を示した後のみ

### 不採用判定

- 修正版 kalman モードが ic モード（現行既定）に OOS で勝てなければ、
  kalman/cs_var モードごと削除候補としてレポート（コード簡素化に寄与）

## A6. マクロ因子感度行列のファンダメンタル導出

### 設計（データ作業が主体、標準モデルで実行可能）

現行 `MACRO_SENS_MATRIX`（USDJPY / CLF / TNX × TOPIX-17）はハードコード。導出型に置換:

| 因子 | ファンダメンタル・プロキシ | データ源 |
|------|--------------------------|---------|
| USDJPY | セクター構成銘柄の海外売上高比率（時価加重） | 有報 XBRL or FactSet相当。取得困難なら東証セクター別統計 |
| 原油 (CLF) | 産業連関表のエネルギー投入係数（セクター写像） | 総務省 産業連関表（公開） |
| 金利 (TNX) | 有利子負債/EBITDA + 金融セクターはNIM感応度を別扱い | 財務データ |

正規化: 各因子列を cross-sectional z-score → `MACRO_SENS_MATRIX_FUNDAMENTAL` として
`macro_sens_matrix: fundamental` の config 分岐に追加（既存 "derived" 分岐と同型）。

### 検証仕様

- 実測感度（ローリング回帰β）とのランク相関 ≥ 0.5 を採用の必要条件とする
- ファンダメンタル版・derived版・ハードコード版の3者で OOS 比較。差が僅少なら
  ハードコード版維持（データパイプライン依存を増やさない）

## A7. ウォークフォワード検証と Deflated Sharpe の標準プロトコル

### 仕様（→ 詳細は C_validation_frameworks.md C1 と統合）

- **分割**: 訓練 expanding（開始 2015-01-05）、テスト1年、purge = corr_window(60) + 1日、
  embargo = 5日。テスト区間は 2018–2026 の年次ロール（9区間）
- **検定**: 各区間の net Sharpe を報告し、全区間統合の Deflated Sharpe Ratio
  （Bailey & López de Prado 2014）で判定。試行回数 N は C1 の trials registry を正本とする
- **感度分析**: 変更パラメータ±20%摂動（各2点）での Sharpe 変化を必ず併記。
  摂動でSharpeが符号反転するパラメータは「脆弱」とフラグ

### この設計の理論的根拠

- 日次リターンの自己相関は小さいが、**パラメータ選択は同一ヒストリーの反復使用**なので
  区間間の独立性が本質的制約。9区間 × 反復試行の多重性を DSR で補正する以外に
  統計的に誠実な方法はない
- purge 幅 60 日は corr_window 由来（訓練末尾の相関窓がテスト区間に浸食しないため）
