# Gap Denominator Floor Tail Risk Experiment Report

## 概要

ギャップ分母フロア `denom = max(1 + gap_filt, 0.1)` のテールリスクについて、フロア引き上げ・シグナルクリップ・フラットポジション化の3つの改善案を現行と比較バックテストした。

## 背景と仮説

現行のgap調整ロジック（`blp_base.py:227`, `signal.py:144`, `sector_relative_ensemble_rrr.py:449`）:

```python
denom = np.maximum(1.0 + gap_filt, 0.1)
signal = (1.0 + r_hat_jp_cc) / denom - 1.0
```

`gap_filt < -0.9` の場合、`denom` が0.1にフロアされ、シグナルが最大10倍に増幅される。これは分布の裾を重くし、テールリスクの源となる可能性があった。

### 修正案

1. **フロア引き上げ**: 0.1 → 0.3 または 0.5（増幅率を最大3.3x / 2.0xに制限）
2. **シグナルクリップ**: signal を ±3% または ±5% でクリップ
3. **フラットポジション**: `denom < 0.5` のティッカーは signal=0 にする
4. **組み合わせ**: Floor=0.3 + Clip ±5%

## 実験設計

- **期間**: 2015-01-05 〜 2026-07-10（2733営業日）
- **モデル**: `SectorRelativeEnsembleBLPEnhancedModel`（V1パス、raw_blpx=0.8 + raw_pca=0.2）
- **コスト**: 片道5bps + 金利・貸株・逆日歩含むnet評価
- **手法**: monkey-patching により `_BLPBase._apply_gap_adjustment` と `signal.compute_signal` の両方をパッチ。バリアント間でグローバルキャッシュ（`_RAW_PCA_CACHE`, `_RESIDUAL_PCA_CACHE`, `_BLP_CORR_CACHE`）をクリア
- **7バリアント**: Baseline / Floor=0.3 / Floor=0.5 / Clip±3% / Clip±5% / Flat(denom<0.5) / Floor=0.3+Clip5%

## 分析

### フロア変更が効かない理由

**`denom < 0.1` は観測期間中0件**であった。`gap_filt` の最小値は -0.723 で、denom の最小値は 0.277。つまり現行のフロア0.1は一度も発動していない。

Floor=0.3 は24件（0.035%）、Floor=0.5 は54件（0.079%）のみ影響するが、これらは17ティッカー中の1-2ティッカー・1-3日程度の極限定的な変更であり、ポートフォリオ全体への影響は検出可能なレベルに達しない。

### Clip ±3% の微小改善

Clip ±3% のみ Sharpe 3.884 → 3.902 (+0.5%)、Kurtosis 6.44 → 5.92 (-8%) の微小改善。Tail1% も -1.290% → -1.275% と改善。これは `gap_filt` とは無関係に、BLPXシグナル自体の極端値（±3%超）を数件クリップした効果。ただし改善幅はノイズレベルであり、過学習リスクを考慮すると採用根拠として弱い。

### なぜテールリスクが顕在化しないか

`gap_filt` は `gap_open_coef * gap_idio + (gap_open_coef - topix_beta_coef) * gap_syst` で計算される。各成分:
- `gap_idio = gap_vec - betas_vec * topix_night_t`（idiosyncratic gap、通常 ±2%以内）
- `gap_syst = betas_vec * topix_night_t`（systematic gap、TOPIX夜間リターン×beta）

`gap_open_coef = 0.70` と `gap_open_coef - topix_beta_coef = 0.10` でフィルタされるため、`gap_filt` は元のgapより大幅に縮小される。結果として `1 + gap_filt` が0.1を下回るには、元のgapが -130% 以上の極端値が必要であり、現実のETFリターンでは発生しない。

## 結論

**不採用**。ギャップ分母フロアのテールリスクは理論的には存在するが、現行データでは顕在化していない。

- **フロア0.1は観測期間中一度も発動していない**（denom最小値 = 0.277）
- Floor引き上げ（0.3, 0.5）は0.035-0.079%の観測にしか影響せず、バックテスト結果に変化なし
- Clip ±3% は微小改善（Sharpe +0.5%, Kurt -8%）だが、ノイズレベルであり過学習リスクを考慮すると採用根拠として弱い
- Flat position化も効果なし（影響件数が少なすぎる）

### 今後の方向性

- フロア0.1は理論上のセーフティネットとして維持で問題ない。実データでは発動しないが、ブラックスワン型の巨大ギャップ（例: 財務ショック、制度変更）に対する防御として意味がある
- もしシグナルの極端値が懸念される場合は、フロア変更ではなく Clip ±3% のようなシグナル側のクリップが Kurtosis 改善に有効。ただし改善幅が微小なため、ウォークフォワード検証で確認が必要
